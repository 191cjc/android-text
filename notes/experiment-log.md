# Experiment Log

## 2026-06-26

- Source page: `https://www.4399.com/flash/115225_2.htm`
- Iframe entry: `https://sbai.4399.com/4399swf/upload_swf/ftp10/honghao/20130530/27/jjxzfcms.htm`
- Main SWF discovered from iframe: `xfbbv451.swf`
- Local copy: `original/xfbbv451.swf`
- Initial SWF signature: `CWS`, version 19, declared uncompressed length `5448442`.
- First Ruffle run loaded the SWF but hit browser CORS for `stat.api.4399.com` and `cdn.comment.4399pk.com`.
- Added Electron request interception that redirects those known auxiliary resource hosts to local `/external`.
- Added `media.5054399.net` after runtime logs showed an ad/cover metadata request.
- Added full request logging to locate remaining runtime failures such as `getServerTime` without broad request mutation.
- Added a narrow read-only proxy policy for `save.api.4399.com/?ac=get_time`; this fixes server-time CORS without enabling local override for save/account APIs.
- Read-save test in local mode reached only `get_time`, not the real read-save API. This indicates missing 4399 page login context rather than a blocked read-save request.
- Added `remote` mode to load the original 4399 page and inject local Ruffle, preserving the site's login/Cookie/UniLogin context.
- Remote mode initially loaded `ruffle.js` but the WASM fetch was blocked by CORS. Added CORS headers to local `/ruffle` assets and restricted injection to the game iframe.
- Remote mode then reached the original SWF URL but Ruffle could not fetch it cross-origin, so the game SWF is redirected to local `/game/xfbbv451.swf` while the page stays on 4399.
- 4399's own Flash fallback UI can still show "missing Flash" after Ruffle injection. In remote mode, `blockflashtip.html` and `flashdist.js` are redirected to local no-op files.
- The remote page later went blank because `/jss/flashopen1.js` detected missing Flash and replaced `#swfdiv`/`#addiv` with the fallback iframe. Remote mode now redirects `flashopen1.js` to `public/noop.js`; screenshot verification shows the game menu renders again.
- Login inside the Electron window succeeds; runtime heartbeats changed from `uid=0` to the logged-in uid. The game then stopped on its own "initializing" screen while calling official save/platform APIs. Added Electron-level CORS exposure for a narrow list of official initialization/read endpoints (`User.Authenticate`, `get_session`, `get_token`, `get_list`, `my.4399.com/services/game-play`, and official read-only total recharge lookup). These calls are not overridden locally and payment/recharge values are not fabricated.
- Read-save now reaches `User.Authenticate`, `get_session`, `get_token`, and `get_list` with real 4399 session cookies attached. The remaining post-login/tutorial hang correlates with Ruffle throwing `PayEvent` type-coercion errors inside 4399's platform control SWF (`usePayApi` / `rechargedMoney`). Upgraded Ruffle from `0.3.0` to `0.4.0-nightly.2026.6.26` to test whether the compatibility issue is fixed without changing game data or recharge values.
- Added `tools/patch-pay-event-listener.js` and generated `modified/xfbbv451.swf`. The patch only changes the main game SWF's `payProcess(e:unit4399.events.PayEvent)` listener parameter type to ABC `any`, so Ruffle can deliver `PayEvent` objects coming from the loaded 4399 control SWF even when they live in a separate ApplicationDomain. This is a local runtime compatibility patch; it does not override recharge APIs, fabricate payment values, or alter item/save data.
- Direct probes showed the discontinued payment-query endpoints now return `game_not_exist` for `GetTotalRecharge`, `GetTotalPay`, and `GetMoney`; this is a normal official dead-end for this stopped game and can trigger the control SWF's error branch during ordinary initialization. Added a local `/offline-4399` compatibility endpoint and Electron redirect for those payment-query reads. It returns an encrypted DES-ECB/Base64 `time####0` payload using the legacy `4399api_` key so the game sees zero recharge/pay balance. `GetToken` receives a local transient token, while payment/consume actions (`Pay`, `Dec`) return unavailable and never simulate success.
- Replaced `modified/local/xfbbv451.swf` with the patched `modified/xfbbv451.swf` to avoid accidentally loading an unpatched main SWF when switching mods.
- Expanded the PayEvent compatibility patch: besides relaxing listener parameter types, the patcher now scans ABC method bodies and replaces `astype/coerce/istype unit4399.events::PayEvent` bytecode checks in the main SWF with `coerce_a` plus `nop` padding. Static verification now shows no PayEvent method params or PayEvent bytecode checks remain in `modified/xfbbv451.swf` or patched `modified/external/.../ctrl_mo_v5.swf`.
- Added local asset/version logging and query-string cache busting for `/game` and `/external` redirects. Runtime logs confirmed the game loads `modified/xfbbv451.swf` and `modified/external/.../ctrl_mo_v5.swf`, yet Ruffle still reports `PayEvent` type coercion during `usePayApi`/`rechargedMoney`.
- Copied all cached 4399 control SWFs (`4399.swf`, `A4399dv_base.swf`, `A4399dv_base_main.swf`, `open4399tools_AS3.swf`) into `modified/external/.../control/` so the control stack is served consistently from local overrides, not mixed remote proxy responses.
- The game appears to be fully discontinued, so normal official initialization/save/payment endpoints are no longer a reliable runtime path. Expanded the local `/offline-4399` compatibility layer to cover the ordinary save API flow (`User.Authenticate`, `get_time`, `get_session`, `check_session`, `get_token`, `get_list`, `get`, `save`) plus `my.4399.com/services/game-play`. Local saves are stored in `data/offline-saves.json`.
- Patched `ctrl_mo_v5.swf` to rename the discontinued payment event names `usePayApi` and `rechargedMoney` to local disabled event names. This prevents Ruffle from crashing on those cross-SWF `PayEvent` dispatches while keeping payment and recharge behavior unavailable; payment queries still return encrypted zero values only.
- Switched strategy away from the official 4399 page/control-stack path. The main `xfbbv451.swf` is mostly a 4399/Flex wrapper and contains a large embedded `DefineBinaryData` SWF named `L4399Main_gamefile`. Exported it to `original/L4399Main_gamefile.swf` and `modified/L4399Main_gamefile.swf`, then made local mode load that SWF directly. Remote mode still uses the old `xfbbv451.swf` wrapper for comparison.
- Added `tools/dump-swf-assets.js` and `npm run dump-assets`. The wrapper SWF exports 11 binary assets, including the embedded game SWF. The embedded game SWF exports 145 image resources and 3 sound resources; JPEG3 alpha data is also preserved separately. Static strings enumerate many runtime external SWFs for skins, equipment, levels, monsters, and UI.
- Direct local launch of `L4399Main_gamefile.swf` starts under Ruffle and reaches the game's own "游戏初始化中" screen with a much shorter request chain. This confirms the new single-player/offline patch target should be the embedded game SWF rather than the official 4399 web page or the outer wrapper.
